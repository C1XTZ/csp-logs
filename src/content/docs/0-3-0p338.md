# Changelog

## New features, options and improvements

*   AI:
    *   Trajectory straightening fixed, shouldn't let AIs to crash into other cars anymore;
    *   Impatient behavior reworked;
    *   `Adjust shifting for slow sectors` option is fixed, no longer active at all times, applies in pits, less aggressive for racing cars;
*   FSR settings are reworked into upscale settings (presets remain compatible):
    *   FSR updated to 1.0.2;
    *   New simple methods: NVIDIA Image Scaling, Snapdragon GSR, Basic (replaces old debugging option);
    *   New Extra FX-based methods: FSR2, FSR3, DLSS, Snapdragon GSR 2, OptiScale (currently, requires `Single screen` render mode);
    *   Advanced upscalers, as well as regular ones with low resolution scale, force virtual mirror to be drawn last;
    *   `Disable for screenshots` option should be working properly;
*   Extra FX: motion blur/TAA no longer freaks out on resizes, few smaller fixes;
*   Adaptive V-sync is readjusted and disabled by default;
*   `Use interior sounds everywhere` tweak rewritten to be compatible with reworked audio system, with more options;
*   Fix for a bug with AC audio playing at 100% volume for a frame shortly after AC has finished loading;
*   Old CSP setting presets with orientation-based culling no longer mess up new KN5 loader;
*   Rain FX: chance of windscreen wipers leaving a trace of dirt greatly reduced;
*   Default Weather FX style: volumetric fog cooperates with rain haze, better fades out when sun is low, replaces fog backlit entirely;
*   A crash in replays with a certain combination of Extra FX settings fixed;
*   CSP skidmarks properly read AO value from local area map;
*   New tool displaying the state of track area maps;
*   Compatibility with Assetto Corsa TAA/DLAA patch by Snowy Moon (AC anticheat a bit too jumpy, please let me know if you'd run into any other integrity check failures);
*   Ctrl+Alt+Shift+Win+F12 option moved to general settings, enabled by default, generates more informative crashes as a way to get more insight into the configuration a user is running;
*   In-game binding controls can use controllers in main menu and such (there was an issue with `controllers integration` tweak moving mouse with controllers);
*   Crash on some car config reloads fixed;
*   Applying `SHADER_REPLACEMENT_…` edits live no longer unloads dynamic car texture entries;
*   Device order fix: a few fixes, as well as a bit of extra logging in an attempt to try and solve some issues;
*   Car controls override: docs fixed (there was a typo in `combineAxes`);
*   `script.asyncUpdate` of Weather FX style is called again (should be working fine now, there was a bit of a regression related to introduced coroutines support in preview302);
